# Implementation Notes – Design System Monorepo

## Naming

- **CSS custom properties:** Prefix `--ds-` (e.g. `--ds-semantic-color-action-primary-default`). Generated by Style Dictionary from W3C DTCG tokens in `@rgrmdesign/ds-tokens`.
- **Component classes:** Prefix `.ds-` with BEM modifiers where needed (e.g. `.ds-button`, `.ds-button--primary`, `.ds-form-field__label`) to avoid collisions with host applications.

## Token mapping

- **Source:** Figma / Tokens Studio export as W3C DTCG JSON in `packages/tokens/src/tokens/`.
- **Pipeline:** Style Dictionary 4 reads DTCG (`$type`, `$value`), transforms to CSS variables with prefix `ds`, and writes `dist/tokens.css` and `dist/tokens.json`.
- **Consumption:** `@rgrmdesign/ds-styles` imports the built tokens CSS; all component CSS uses `var(--ds-components-*)` and shared styles use `var(--ds-semantic-*)` / `var(--ds-primitives-*)`. No duplicate token values in styles.

## State handling

- **Prefer attributes:** `[disabled]`, `[aria-invalid="true"]` so state is both semantic and styled without extra classes.
- **When needed:** Use state classes (e.g. `.is-loading`) for states that don’t have a native attribute (e.g. loading). Button uses both `aria-busy` and `.is-loading` for clarity.

## Accessibility (baseline)

- **Focus:** All interactive elements use `:focus-visible` and token-based outline/ring (e.g. `--ds-semantic-focus-ring-width`, `--ds-semantic-focus-ring-offset`) for WCAG-friendly focus indication.
- **FormField:** Label is associated via `htmlFor` and input `id`; help and error have stable `id`s and are referenced by the input’s `aria-describedby`; error state sets `aria-invalid="true"` on the input; error message has `role="alert"`.

## Single source of truth

- **@rgrmdesign/ds-styles** owns all component CSS (classes and states) and consumes only `@rgrmdesign/ds-tokens` output.
- **@rgrmdesign/ds-components-react** and **@rgrmdesign/ds-components-html** use the same `.ds-*` classes and do not duplicate styling logic. React components apply classes and ARIA attributes; HTML snippets are copy-paste markup using the same classes.

## Dependency direction

```
@rgrmdesign/ds-tokens → @rgrmdesign/ds-styles → @rgrmdesign/ds-components-react
                                        → @rgrmdesign/ds-components-html
                                        → apps/storybook
```

No reverse dependencies. Storybook depends on tokens, styles, and React components for documentation and examples.
